二、产品概述

2.1 产品定位

牌组管理系统是塔罗笔记应用的核心模块，用于管理用户的实体塔罗牌收藏，记录牌组使用档案，建立与牌灵的深度连接。

2.2 核心目标

  1. 实体牌组数字化 - 上传牌组图片，建立电子档案
  2. 牌灵绑定与管理 - 每副牌绑定一张大阿卡纳作为牌灵
  3. 使用记录追踪 - 记录每次使用牌组的抽牌案例
  4. 多维度关联 - 与牌灵、客户、抽牌记录形成数据网络

2.3 用户场景

场景1：新用户添加第一副牌

小美刚买了一副伟特塔罗，拍照上传到系统，选择"愚人"作为牌灵，系统自动标记为"第1副牌"。

场景2：记录与牌的对话

每天抽牌前，小美会和牌组"对话"，记录牌组给她的感受和建议，形成独特的"牌组日记"。

场景3：查看使用统计

一个月后，小美发现"第1副牌"使用频率最高，主要用于感情咨询；"第2副牌"（托特）主要用于自我探索。

场景4：低频使用提醒

系统提醒小美"第3副牌"已经10天没用了，她决定今天用这副牌给自己抽一张。

3.2 功能模块详细说明

3.2.1 牌组上传模块

功能描述: 允许用户上传自己的实体塔罗牌组，建立电子档案

核心流程:

添加牌组
├── Step 1: 上传图片
│   ├── 牌盒封面图（必填）
│   ├── 牌背照片（选填）
│   └── 样张（选填，最多5张）
│
├── Step 2: 填写信息
│   ├── 牌组名称（必填）
│   ├── 购买日期（必填，用于生成编号）
│   ├── 牌数（必填，默认78，可修改）
│   ├── 品相（单选：全新/良好/一般/磨损）
│   └── 备注（选填）
│
└── Step 3: 选择牌灵
    ├── 展示22张大阿卡纳
    ├── 支持搜索和筛选（元素）
    └── 支持随机抽取

自动生成编号规则:

  • 根据购买日期排序
  • 最早购买的为"第1副牌"
  • 编号格式："第N副"
  • 编号不可手动修改

3.2.2 牌灵绑定模块

绑定规则:

规则项 说明
绑定数量 1副牌组 = 1张牌灵
更换规则 原则上不更换，支持"换主人"重新绑定
绑定方式 主动选择 / 随机抽取
数据关联 与牌灵总览的22张牌数据关联

3.2.3 使用记录模块

记录类型:

类型 说明 内容
与牌对话 主观交流记录 问题、感受、抽到的牌
抽牌案例 客观服务记录 客户、问题、牌阵、结果

3.3 筛选与搜索

筛选条件:

筛选类型 选项 说明
牌灵标签 全部/愚人/魔术师/女祭司/... 按牌灵筛选
使用频率 常用/偶尔/很少/未使用 按使用次数
品相 全部/全新/良好/一般/磨损 按品相状态

5.2 设计原则

  • 与现有设计保持一致 - 复用项目已有的视觉风格和交互原则
  • 横向滑动展示 - 全部牌组采用横向滑动，+号在最左侧
  • 牌灵标签筛选 - 筛选条件使用牌灵标签而非分类
  • 自动生成编号 - "第N副牌"按购买日期自动生成
  • 模块化内容 - 牌面展示独立为子页面，避免详情页过长

六、技术实现

6.1 存储架构

采用离线优先 + 云端同步架构：

前端 (Browser)
├── Memory Cache (运行时缓存)
├── IndexedDB (本地持久化)
└── Supabase Client (云端同步)

同步策略：
1. 读取：Memory → IndexedDB → Supabase
2. 写入：Memory → IndexedDB (同步) → Supabase (异步)

6.2 API 接口

// 牌组管理
GET    /api/decks                    // 获取所有牌组
GET    /api/decks/:id                // 获取单个牌组
POST   /api/decks                    // 创建牌组
PUT    /api/decks/:id                // 更新牌组
DELETE /api/decks/:id                // 删除牌组
PATCH  /api/decks/:id/pin            // 置顶/取消置顶
PATCH  /api/decks/:id/spirit         // 更换牌灵

// 使用记录
POST   /api/decks/:id/usage          // 记录使用
GET    /api/decks/:id/usage          // 获取使用记录

// 对话记录
POST   /api/decks/:id/conversation   // 添加对话
GET    /api/decks/:id/conversation   // 获取对话记录

// 统计数据
GET    /api/decks/:id/stats          // 获取统计
GET    /api/decks/:id/trend          // 获取趋势
