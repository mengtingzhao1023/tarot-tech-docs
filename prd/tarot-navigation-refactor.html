塔罗牌区域导航重构 PRD

从侧边栏导航改为顶部导航栏模式，提升用户体验

文档版本 v1.0
创建日期 2026-04-12
负责人 总工程师
文档状态 待审核

一、项目背景

1.1 现状问题

当前塔罗牌区域采用侧边栏导航，包含以下问题：

  • 导航层级过深，用户操作路径长
  • "塔罗管理"作为侧边栏主标题，与整体导航风格不统一
  • 牌阵库、牌背库等功能隐藏在二级菜单中，不够直观

1.2 目标

将塔罗牌区域从侧边栏导航改为顶部导航栏模式，参考客户管理区设计，提升用户体验和操作效率。

二、需求概述

2.1 核心需求

  1. 移除侧边栏导航，改为顶部横向导航栏
  2. 导航项支持下拉展开子菜单
  3. 保持与客户管理区一致的视觉风格
  4. 面包屑导航相应调整

2.2 用户场景

  • 场景1: 用户想查看牌阵库 → 点击顶部"牌阵库"标签直接跳转
  • 场景2: 用户想管理牌组 → 点击"我的牌组"下拉展开子项选择
  • 场景3: 用户想查看牌组档案 → 通过"我的牌组"→"牌组档案"进入

三、详细设计

3.1 导航结构

顶部导航栏（横向标签式）

┌─────────────────────────────────────────────────────────────────┐
│  [牌林总览]  [我的牌组 ▼]  [牌背库]  [牌阵库]                   │
│              ├─ 牌阻挡                                          │
│              ├─ 常用牌组                                        │
│              └─ 牌组档案                                        │
└─────────────────────────────────────────────────────────────────┘

导航项说明

导航项 类型 目标页面 说明
牌林总览 一级标签 tarot-cards.html 22张大阿卡纳牌展示
我的牌组 下拉菜单 tarot-mine.html 默认展示我的牌组主页面
├─ 牌阻挡 子项 tarot-deck-blocks.html 物理牌组管理（新增页面）
├─ 常用牌组 子项 tarot-common.html 常用牌组展示
└─ 牌组档案 子项 tarot-deck-archive.html 牌组使用档案（新增页面）
牌背库 一级标签 tarot-back.html 牌背样式管理
牌阵库 一级标签 tarot-spread-home.html 20个预设牌阵+自定义

3.2 面包屑调整

调整前:

塔罗管理 / 塔罗牌库

调整后:

塔罗牌管理 / 牌林总览
塔罗牌管理 / 我的牌组
塔罗牌管理 / 我的牌组 / 牌组档案
塔罗牌管理 / 牌背库
塔罗牌管理 / 牌阵库
