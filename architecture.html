L1 规则层

工程规范制定、代码标准定义、接口契约设计

在塔罗项目中：制定了前端组件规范、API接口规范、命名规则

L2 记忆层

Skill上下文管理、对话历史记录、项目记忆沉淀

在塔罗项目中：使用tarot-memory skill记录项目上下文

L3 模板层

代码模板库、PRD模板、设计模式库

在塔罗项目中：建立了页面模板、组件模板、PRD模板

L4 架构层

双层目录结构、模块划分、依赖管理

在塔罗项目中：外层约束层+内层执行层的双层架构

L5 同步层

双设备同步、数据备份、版本控制

在塔罗项目中：IndexedDB + Supabase双路径存储

L6 调度层

并行开发调度、任务分配、进度跟踪

在塔罗项目中：多AI角色并行开发，前后端分离

L7 质检层

双层审查机制、代码审查、质量把关

在塔罗项目中：分项自查+全局终审的双层审查

L8 安全层

双路径备份、数据安全、访问控制

在塔罗项目中：本地存储+云端同步双保险

L9 归档层

知识沉淀、文档归档、经验总结

在塔罗项目中：完整的PRD文档库和技术文档

L10 交付层

版本收敛、发布管理、交付验收

在塔罗项目中：模块化交付，逐步上线

双层架构分工

全局约束层与执行开发层的职责分离

🎯

外层全局约束层

总工程师AI负责

  • • 需求定稿 - PRD文档审核，功能边界确认
  • • 规范制定 - 工程标准、接口契约定义
  • • 技术选型 - 开源方案检索，复用策略
  • • 任务拆分 - 角色分配，并行策略
  • • 全局终审 - 代码规范、联动一致性
  • • 版本收敛 - 模块冻结，交付把关
⚡

内层执行开发层

各端工程师AI负责

  • • 独立开发 - 按接口契约并行编码
  • • 规范遵循 - 严格执行全局标准
  • • 单元测试 - 模块级功能验证
  • • 分项自查 - 代码质量、文档完整
  • • 接口对齐 - 前后端契约确认
  • • 记忆更新 - 每10轮更新Skill
